## Summary

- show a quota upgrade overlay for authenticated free users after the 10-conversion weekly limit is exhausted
- include plan and support links in quota-exceeded responses for both direct `/convert` and streaming upload flows
- keep the anonymous signup/login quota lock unchanged while reusing the same frontend overlay structure
- add regression coverage for HTTP and SSE quota-exceeded payloads

## Why

Authenticated free users were only seeing a generic quota error after hitting the weekly 10-conversion limit. They did not get a clear upgrade path to the plans page or a support fallback, which blocked the intended commercial flow.
