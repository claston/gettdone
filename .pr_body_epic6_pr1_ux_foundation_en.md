## Summary

- Rebuilds the frontend visual foundation with a service-oriented UX (not SaaS-oriented).
- Applies a new design system baseline (surface hierarchy, typography, spacing, premium motion, responsive layout).
- Refactors the single-page flow into clear steps: upload, processing, and review/download.
- Keeps current reconciliation functionality and DOM integration intact while improving copy and interaction feedback.

## Why

The previous UI was built mainly for functional validation. For EPIC 6, we need a stronger UX baseline that reduces friction and communicates the product as a service workflow (no login, no pricing/paywall patterns for validation stage).
