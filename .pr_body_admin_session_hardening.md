## Summary

- replace long-lived admin bearer tokens with short-lived session cookies issued by `/admin/auth/login`
- remove admin query-string auth and the legacy `PLANS_ADMIN_TOKEN` / `/plans/activate` path from the admin module
- update admin frontend flow to use cookie-based sessions, refresh on `401`, and keep localhost dev cookies on the same host
- expand admin, plans, checkout, and session tests to cover the new contract and dev cookie defaults

## Why

The admin area was still using a durable signed user token as its main credential, and several admin endpoints accepted credentials through query parameters or a shared legacy environment token. That made admin access harder to revoke safely and increased leakage risk through logs, URLs, and copied requests. This change moves admin onto the existing revocable session model and tightens the auth surface without changing the anonymous conversion flow.
